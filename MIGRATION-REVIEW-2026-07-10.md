### Hard violations

1. **`docusaurus.config.ts` — link validation not strict**
   - **Standard:** `copilot-instructions.md` §2.2 states `onBrokenLinks` is set to `'throw'`.
   - **Finding:** `docusaurus.config.ts` sets `onBrokenLinks: 'warn'`.
   - **Impact:** Broken links will not fail the build and may go unnoticed.

2. **`sidebarsOgs.ts` uses explicit doc IDs instead of autogenerated categories**
   - **Standard:** `copilot-instructions.md` §Navigation says “Use `type: 'autogenerated'` for directories to simplify maintenance.”
   - **Finding:** Every entry in `sidebarsOgs.ts` is an explicit doc ID (`tools/README`, `tools/openprotocol/nexo`, etc.).
   - **Impact:** New files added to the file system will not appear in the sidebar automatically.

3. **Absolute internal links in migrated Markdown**
   - **Standard:** `copilot-instructions.md` §Linking says “Never use absolute paths (`/docs/...`) as mount points differ between projects.”
   - **Findings:**
     - `ogs-docs/appnotes/station_io.md:31` uses `/ogs/tools/openprotocol/`
     - `ogs-docs/tools/positioning/README.md:140` uses `/ogs/v3/lua/customtools/`
   - **Impact:** Links will break if the `baseUrl` or mount point changes.

4. **Forbidden `id` frontmatter in migrated docs**
   - **Standard:** `copilot-instructions.md` §Frontmatter forbids `id` and `slug` in shared files (and by extension in any mounted docs) to avoid ID collisions.
   - **Findings:**
     - `ogs-docs/tools/misc/hikrobot-vm.md` declares `id:`
     - `ogs-docs/tools/openprotocol/mtc.md` declares `id:`
     - `ogs-docs/tools/openprotocol/pf6-pf8.md` declares `id:`
   - **Impact:** Potential doc ID collisions when content is shared or mounted in another Docusaurus instance.

### Judgement calls / notes

5. **`sidebars.ts` uses explicit `shared/...` IDs**
   - This follows the “prefix with `shared/`” rule, but it is not autogenerated. Acceptable for curated ordering, though not the stated preference.

6. **Absolute paths in `src/pages/index.tsx` and `docusaurus.config.ts` navbar/footer**
   - The `to:` values use absolute paths. The standard’s wording targets Markdown links, so these are flagged as judgement calls. They work with the current `baseUrl: '/Docusaurus/'` but could break if the mount point changes.

7. **React 19 with Docusaurus 3.9.2**
   - `package.json` uses React 19. Docusaurus v3 does not officially support React 19. This is a compatibility risk, not a documented project standard violation.
